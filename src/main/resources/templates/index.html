Gerenciar Clientes

O CRUD de Clientes seria similar ao de Pessoas, mas com uma etapa adicional de **vincular uma Pessoa já cadastrada** para criar a entidade Cliente (tb_cliente, que tem a FK para tb_pessoa).

Estrutura: Tabela de Clientes + Formulário de Criação/Edição.

Gerenciar Funcionários

O CRUD de Funcionários utilizaria a mesma lógica de Contratos:

  1. Seleção de uma Pessoa (FK para tb_pessoa).
  2. Campos específicos: Data Admissão, Cargo, Salário.
  3. O campo `fk_id_contrato` pode ser nulo ou servir para vincular o funcionário ao contrato de trabalho, dependendo da regra de negócio (aqui mantemos simples e focaríamos nos campos principais).
